# 工具类页面 UI 模板

本项目按纯工具型应用处理，不使用营销页或内容站首页结构。

## 页面骨架

```text
顶部应用导航
筛选/搜索工具条
工具列表或工具主体
必要状态反馈
```

## 导航

- 顶部导航只放高频入口：全部工具、一级分类、管理。
- 一级分类用于回到首页并筛选工具。
- 二级菜单只展示工具名称，不展示说明文案。
- 技术接口、共享数据等入口不放在普通用户主导航里。

## 首页

- 首页是工具目录，不做 hero。
- 首屏优先展示搜索、分类和工具列表。
- 工具卡片保持紧凑，显示名称、简短说明、分类和能力标签。
- 不使用大面积装饰背景、渐变光斑或营销式口号。

## 工具页

- 工具页首屏应直接进入可操作区域。
- 标题区保持紧凑，仅保留工具名、必要说明和关键操作。
- 表单、编辑器、预览、导出等能力优先于介绍文案。
- 同类表单工具应复用一致的字段密度、按钮位置和结果区结构。
- 每个工具页必须显示当前版本号，当前由 `public/site-nav.js` 自动注入顶部版本标识和工具页底部版本页脚。
